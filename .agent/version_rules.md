# AI バージョン命名ルール (Version Rules)

プロジェクトやリリースのバージョン番号の付け方に関するルールです。

## 1. バージョン形式
原則として **セマンティック バージョニング (Semantic Versioning)** を採用します。
形式: `X.Y.Z`

- **X (Major)**: 互換性のない大きな変更、またはプロジェクトの大きな節目。
- **Y (Minor)**: 互換性のある機能追加。
- **Z (Patch)**: 互換性のあるバグ修正や軽微な調整。

## 2. バージョンアップの判断
- **メジャーアップデート**: プロジェクトのフェーズが大きく変わる際、またはアーキテクチャの抜本的な変更時。
- **マイナーアップデート**: 新機能の追加、新しい画面やコンポーネントの追加時。
- **パッチアップデート**: 文言修正、既存機能の細かな修正、依存ライブラリの更新時。

## 3. 記録場所
- `AI/version_control.md` または `package.json` 等のプロジェクト構成ファイルに現在のバージョンを明記します。
