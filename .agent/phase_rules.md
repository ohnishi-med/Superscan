# AI フェーズルール (Phase Rules)

プロジェクトのライフサイクルを管理するためのフェーズ定義と運用ルールです。

## 1. フェーズの定義
プロジェクトを以下の標準的なフェーズに分割します：
- **企画 (Plannig)**: 要件定義、基本設計、タスクの分解。
- **プロトタイプ (Prototyping)**: 最小機能の実装と技術的な実現性の検証。
- **開発 (Development)**: 本実装、機能拡充。
- **検証 (Verification)**: テスト、品質確認、バグ修正。
- **完了 (Closed)**: リリース準備完了またはアーカイブ。

## 2. フェーズ管理ドキュメント
- 各プロジェクトの `Documents/phase_management.md` で現在のフェーズを追跡します。
- 現在のフェーズの目標と、完了した項目の履歴を維持します。

## 3. フェーズの移行
- 設定したマイルストーンを達成した際、AIにフェーズ移行を宣言させ、ドキュメントを更新します。
- 次のフェーズの目標を明確にし、タスクリストをリセットまたは大規模更新します。
