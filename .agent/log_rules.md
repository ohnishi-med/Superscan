# AI 作業ログルール (Log Rules)

日々の作業記録（日報・ログ）に関するルールです。

## 1. ログの目的
- その日の作業内容、進捗、直面した課題を記録し、AIと開発者のコンテキストを共有します。
- 作業の連続性を確保します。

## 2. 記録場所
- 原則として、各プロジェクトの `AI/daily_logs.md` または `AI/work_log.md` に追記します。

## 3. 記述内容
毎回の記録に以下の要素を含めます：
- **日付**: YYYY-MM-DD
- **本日の目標**: 何を達成しようとしたか。
- **実施内容**: 実際に行った作業（箇条書き）。また、`Documents/environment_management.md` に基づき、環境別の設定項目（DB、APIキー等）に変更を加えたか、または同期が必要な状態かも記録する。
- **課題・保留事項**: 解決できなかった問題や、明日以降に回すタスク。
- **次のステップ**: 次回作業を開始する際の入り口。

## 4. 更新タイミング
- 1日の作業の終わり、または大きなタスクの区切り。

## 5. 環境同期の原則
ブランチ間をまたぐ作業やデプロイ前には、必ず `Documents/environment_management.md` のチェックリストを更新・確認してください。
