# AI ドキュメントルール (Documentation Rules)

プロジェクトの透明性と進捗を管理するためのドキュメント運用ルールです。

## 1. Markdown ファイルの管理
- ドキュメントはすべて Markdown (`.md`) 形式で作成します。
- 階層構造を意識した見出し、リスト、コードブロックを適切に使用し、可読性を高めます。

## 2. フォルダの使い分け
- `Application/`: 開発コード。
- `Documents/`: ユーザー向けの設計書や進捗報告。
- `LandingPage/`: 製品紹介ページ。
- `Progress/`: 進捗管理データ。
- `Project AI Rules/`: (プロジェクト固有) そのプロジェクト専用のAI指示。
